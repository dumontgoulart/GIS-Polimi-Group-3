Project info

A detailed report of our working scheme and the challenges we faced

Project aim

We collected data to create a website (hosted offline in a local server) and a web map, which allows the user to access specific data related to the city of Milan. The project is part of the course of Geographic Information Systems, taught by professor Maria Antonia Brovelli at Politecnico di Milano.

Work flow

The project was divided in two main parts. The first one was to collect the land cover data, which is the physical material at the surface of the earth, you can check the different categories here. The tool used for this was the Land Cover Collector mobile app, created at Politecnico di Milano. To collect the data, we went to Milan zone 7, the area assigned to the group, to geocach our 60 relevant points. Each group member reached 20 points, taking specific pictures about the area and its surroundings. Then, we defined the type of surface according to the land cover categories and the degree of confidence in its classification. After collecting all the points, they were stored into a server.

The second part was to code both the website and the web map. The method to create the web map was based on the practical classes given by the PhD. Candidate Candan Eyl√ºl Kilsedar. For this, some tools were required. First, OSGeo Live, a system developed by The Open Source Geospatial Foundation (OSGeo), was installed. I's a Virtual Machine based on Lubuntu allows to try a wide variety of free and open source geospatial software without installing anything. The second tool used was GeoServer, an open source server for sharing geospatial data. With the Geoserver it was possible to upload the different maps we needed to upload for the web map. Finally, the code was designed using Sublime and it was programmed in HTML, JavaScript and CSS.

The library used for the web map was OpenLayers, which is an open-source JavaScript library for displaying map data in web browsers as slippy maps. For the web map, it was required to create a map with three different basemaps, controls for the map itself, like fullscreen, overlayer and zoom buttons. In addition, the GlobeLand30 map, which is a raster file, was added as an overlayer, showing the categories of cover land for the city of Milan and its legend.

Challenges

The biggest problems we had during the project were: 1) Physical and weather adversities for points that were in difficult spots and far away; 2) There were some technical issues with the server of the app, meaning that two members of the group had to repeat the collecting process in two separated days; 3) The getfeatureinfo function was a bit problematic and there was a considerable effort to make it work smoothly adding it to a popup, but it was not successful.

Website design

This website is based on a template from website html5up.net that was re-designed to adapt it to our needs, altering the code and its content.
